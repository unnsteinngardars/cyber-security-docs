⚡Why memory artefacts are unique

Memory artefacts are the only place certain facts ever exist. The disk can tell you a process ran (Prefetch §10, AmCache §10); only memory can tell you what arguments it ran with, which sockets it had open, and which credentials were resident in LSASS when you walked in. RFC 3227's order of volatility is therefore an artefact-availability rule, not a procedural nicety.

The forensic-value lens for this section: which container preserves which fact, and for how long. A live RAM dump is the gold standard; hiberfil is a free historical snapshot; pagefile is fragmentary but often saves the day when both are missing.

💽RAM dump (raw) — the live snapshot
Process tree Credentials Network Live state

Where it lives: a flat .raw / .bin / .mem / .dmp file roughly the size of installed physical memory.

What it carries — facts that exist nowhere else:

  • Process & thread structures (EPROCESS, ETHREAD) — full process tree with PPIDs, command lines, environment variables, security tokens.
  • Loaded modules & injected code — reflectively loaded DLLs, hollowed processes, unbacked executable pages.
  • Live TCP/UDP endpoints — owning PID, local/remote IPs, state — including connections that were torn down before any disk artefact recorded them.
  • LSASS memory — NTLM hashes, Kerberos tickets, sometimes plaintext credentials.
  • Open handles & mapped files — exactly which registry keys, mutexes, files each process held.
  • Encryption keys — BitLocker FVEK, VeraCrypt master keys, SSH-agent keys, browser cookie-encryption keys.
  • Console history, clipboard, kernel callbacks, rootkit hooks.

Limits: evaporates at power-off. The dump is also a moment — anything paged out at that instant is in pagefile.sys, not here.

📦RAM dump (AFF4 bundle)
Integrity Bundling

Where it lives: single .aff4 container (WinPmem default).

What it carries — same as the raw dump, plus:

  • Embedded pagefile.sys and hiberfil.sys streams so Volatility's --swap-locations resolves paged-out memory without external files.
  • Per-chunk SHA-256 + acquisition manifest (host name, tool version, page-table base, timestamp).
  • A copy of the capturing binary so reproducibility holds across years.

Forensic value vs raw: turns the dump into a self-contained, self-verifying piece of evidence. Export raw later for tools that don't speak AFF4.

Limits: some Volatility plugins still want raw — keep an export beside the AFF4.

🛏hiberfil.sys — historical RAM, free on disk
Past process tree Historical Off-host recovery

Where it lives: C:\hiberfil.sys — hidden, system-protected, locked while running.

What it carries: a Xpress-Huffman-compressed snapshot of physical memory written at hibernate (or partial hibernate via Fast Startup). After decompression it is functionally a RAM dump from the moment the user hit suspend.

Forensic value:

  • A second time-point of memory, often days or weeks before the incident — process tree at suspend, sockets at suspend, credentials at suspend.
  • Fast Startup (default on Win10/11 laptops) writes a partial hiberfil even when the user thinks they never hibernate. The off-host responder almost always has one available.
  • Magic HIBR = unread (full snapshot). Magic RSTR = partially overwritten on resume — still extractable, just degraded.

Limits: compressed and structured — needs hibr2bin or Volatility's hibernation layer to flatten. After a clean resume the kernel sometimes zeroes the file; check magic bytes first.

📄pagefile.sys — fragments after eviction
Strings Creds in plaintext Long-lived

Where it lives: C:\pagefile.sys (or wherever the system was configured to put it).

What it carries: 4 KB pages of process memory the kernel decided to evict to disk. No headers, no index — just a soup of pages whose owning processes may no longer exist.

Forensic value:

  • Plaintext fragments survive long after the source process exited: browser tabs, decrypted Office documents, IM chat buffers, PowerShell command strings.
  • Strings / regex carving recovers URLs, file paths, IP addresses, base64 blobs, credentials.
  • Volatility 3 can attach it (--swap-locations) to resolve memory regions that were paged out at dump time.

Limits: no structure — you cannot reconstruct a process from pagefile alone. Wiped on shutdown only if ClearPageFileAtShutdown=1 (rare).

📃swapfile.sys — UWP / app-suspend swap
App state

Where it lives: C:\swapfile.sys.

What it carries: compressed working sets of suspended Store / UWP / modern apps. Much smaller than pagefile, with a different eviction trigger (app suspend, not memory pressure).

Forensic value: sometimes preserves the in-memory state of UWP apps (Mail, Calendar, OneNote modern, Edge tabs) that were running but suspended at acquisition time. Often overlooked.

Limits: smaller corpus, compressed — useful in conjunction with pagefile, not as a sole source.

🖥.vmem / .vmsn — virtual-machine equivalents
Process tree No driver needed

Where it lives: alongside the VM's .vmx — typically guest.vmem (RAM) plus guest.vmsn (snapshot file holding CPU/register state) for VMware. Hyper-V's .bin + .vsv serve the same role.

What it carries: functionally identical to a live RAM dump — same EPROCESS / sockets / handles — but produced by pausing the VM rather than loading a kernel driver in the guest.

Forensic value:

  • Acquisition is host-side and leaves zero footprint in the guest — ideal when anti-forensic malware watches for driver loads.
  • .vmsn snapshots can capture a known-good baseline plus an incident state, enabling diff analysis.
  • Volatility / MemProcFS treat .vmem like raw memory — no conversion needed.

Limits: only available if the target is virtualised. Some hypervisors compress or encrypt the snapshot file — check the format before parsing.

Cross-references for corroboration

Memory artefacts are most powerful when triangulated against disk-side artefacts:

  • Live process list ↔ Prefetch / AmCache (§10) — an EPROCESS for a binary with no .pf file = either a brand-new execution (no prefetch yet) or anti-forensic cleanup.
  • Open sockets ↔ Event Log Sysmon 3 / Security 5156 (§20) — live sockets without a matching firewall log = possible tampering or kernel-mode networking.
  • Hiberfil process tree ↔ disk $MFT (§04) — a process running in hiberfil whose image file no longer exists on disk = wiped post-hibernate.
  • LSASS credentials ↔ SAM (§07) / SECURITY — plaintext or NTLM hashes recovered from RAM should match (or expose) the on-disk credential material.
  • Pagefile strings ↔ Browser history (§11/browser) / Email PST (§17) — a URL fragment in pagefile can be tied to a specific browsing event or message body.
  • Memory dump timestamp ↔ Disk image acquisition timestamp (§01) — RAM must precede disk, otherwise order-of-volatility was violated.
